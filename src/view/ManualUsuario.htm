1.   Introducción

El proyecto HPC Administration es una aplicación Web que permite la configuración, control y administración de experimentos de forma gráfica y con acceso remoto vía Web. Fue desarrollado para los investigadores, profesores de computación y estudiantes que utilizan el clúster del CIC del Instituto Tecnológico de Costa Rica, teniendo en cuenta la necesidad de administrar y ejecutar experimentos en el clúster sin conocimientos de comandos de consola y desde largas distancias,

Dentro del contexto de este proyecto, un experimento está definido como la ejecución de un programa o script, con una configuración previamente definida, que permita al usuario obtener la salida del programa y almacene estadísticas sobre uso de los recursos de hardware. HPC Administration ayuda a realizar dichas funcionalidades en un ambiente más amigable para los usuarios del clúster, que el acceso al clúster sea Web y que se brinde información valiosa a los usuarios para que puedan optimizar el programa ejecutado. De esta manera, cualquier usuario, sin importar sus conocimientos en la consola de comandos, podrá utilizar el clúster de manera remota.

El presente documento busca describir los principales objetivos del sistema, los elementos de la interfaz gráfica, el ingreso al sistema, las funcionalidades ofrecidas por él y orientar al usuario en el uso de HPC Administration.

 

 

 

2.   Objetivos del Sistema

  •  Capacidad de instalar programas en el clúster. El sistema se encarga de desempacar, descomprimir,  y compilar archivos recibidos por el usuario y guardarlos en el sistema de archivos.
  •  Guardar archivos ejecutables en el clúster. El sistema puede almacenar cualquier archivo binario recibido por el usuario.
  •  Configurar y ejecutar experimentos en el clúster. El sistema puede recibir parámetros para la ejecución de un experimento y empezar la ejecución del mismo. Esta configuración contempla parámetros de paralelismo al igual que parámetros de entrada para el programa sobre el que se realiza el experimento.
  •  Administrar la ejecución de experimentos. El sistema ofrece al usuario la posibilidad de reiniciar, finalizar o detener un experimento durante su ejecución.
  •  Obtener datos de salida de la ejecución de experimentos.
  •  Obtener estadísticas de experimentos. El sistema debe realiza estadísticas por experimento sobre tiempo de ejecución, memoria utilizada y carga del sistema. Estas estadísticas contemplan el paralelismo del clúster.

 

 

3.   Descripción funcional del sistema

El proyecto HPC Administration es una aplicación Web que permite la configuración, control y administración de experimentos de forma gráfica y con acceso remoto vía Web. Para ingresar programas a ejecutar dentro del HPC Administration se puede utilizar las funcionalidades instalar programa, que consiste en desempacar, descomprimir,  y compilar archivos recibidos por el usuario o de un repositorio, o subir ejecutable que permite el ingreso de cualquier archivo binario recibido por el usuario. El sistema utiliza los programas ingresados para la configuración y ejecución de experimentos en el clúster. La configuración de un experimento sirve para definir los parámetros de ejecución del programa y contempla parámetros de paralelismo al igual que parámetros de entrada para el programa sobre el que se realiza el experimento. Una vez en ejecución el experimento, este puede ser detenido y reiniciado por el usuario en cualquier momento. Cuando un experimento finaliza se generan datos de salida de la ejecución del programa con los parámetros de configuración del experimento, además de estadísticas de hardware del experimento como tiempo de sistema para el procesador, memoria utilizada y carga del sistema. Es importante notar que esto solamente ocurre cuando el experimento finaliza exitosamente y que estos datos no son generados si el usuario detiene la ejecución del experimento.

4.  Elementos de Interfaz

Una pantalla está compuesta de una serie de elementos que tienen una función y apariencia física distinta. A continuación se describen los elementos de interfaz presentes en las formas que componen el Sistema.

Elemento

Descripción

Imagen de ejemplo

Botones de acción

Utilizados generalmente al final de los formularios de cada página; permiten llevar a cabo la acción de la transacción. Algunas de las funciones para las que se utilizan son: instalar programa, subir ejecutable, crear experimento, autenticarse en el sistema e iniciar o detener la ejecución de un experimento

Utilizados para que el usuario pueda escoger entre múltiples opciones ofrecidas. Algunos sitios donde se utilizan son para: el tipo de repositorio para instalar un programa, el ejecutable utilizado para la configuración de un experimento y el tipo de parámetro de entrada de un experimento.

Check box

Se utiliza únicamente para definir si se quiere configurar un experimento para ejecución paralela o no. Se usa en la creación de un experimento.

 

5.   Ingreso al Sistema

Cuando el usuario ingresa a la página y se desea autenticar en el sistema, se visualizará la pantalla que se muestra en la figura 5.2:

Figura 5.1 Login

 

El objetivo de la misma es autenticar al usuario para que este pueda utilizar HPC Administration. Para esto se debe ingresar el login del usuario y la contraseña que le permite autenticarse dentro del sistema. Una vez que haya escrito esta información en los campos de texto, debe hacer clic en el botón Aceptar para ingresar en el sistema. Es importante destacar que no hay una forma de que el usuario pueda registrarse por su propia cuenta. Su usuario de sistema debe ser registrado por algún otro usuario administrador.

 

Cuando el usuario se acaba de autenticar en el sistema o cuando se va a Mis experimentos desde el panel de navegación, se visualizará la pantalla que se muestra en la figura 5.2:

Figura 5.2 Mis experimentos

 

El objetivo de la misma es mostrar todos los experimentos configurados y creados por el usuario y que el usuario puede ejecutar.

En la tabla en que se muestran los experimentos del usuario se muestra el nombre del experimento, una pequeña descripción de este, el nombre de ejecutable o programa sobre el que se ejecuta el experimento, una opción para descargar los resultados y estadísticas de la última ejecución del experimento, y un link de navegación para observar el resto de información configurada para el experimento (en Ver detalles). Desde esta última opción es que se navega a la pantalla desde la cual se puede ejecutar el experimento y descargar otros resultados o estadísticas.

 

Cuando se va a Instalar programa desde el panel de navegación, se visualizará la pantalla que se muestra en la figura 5.3:

Figura 5.3  Instalar programa

 

El objetivo de la misma es proveer un medio para que el usuario brinde un programa sin compilar para que el sistema lo compile y desempaque.

Para esto se debe ingresar una descripción del programa, nombre de la carpeta en que se quiere guardar el programa, y hay que indicar si el programa será obtenido de la dirección de un repositorio o de un archivo brindado por el usuario. Si es de un repositorio es necesario ingresar el URL del repositorio en el campo de texto, además de seleccionar por medio del combo box el tipo de repositorio desde donde se obtiene el programa: SVN o gitHub. Si es por medio de un archivo, el archivo ingresado debe ser de la extensión tar.gz. Es importante destacar que los programas deben tener un make que permita la compilación de los archivos. Por último, se debe ingresar el nombre de la carpeta en que se quiere contener el programa; si está última existe para algún programa previo, se sustituirán los contenidos de la carpeta anterior por los de este nuevo programa. Esto  incluye carpetas de subir ejecutables.

 

Cuando se va a Subir ejecutable desde el panel de navegación, se visualizará la pantalla que se muestra en la figura 5.4:

Figura 5.4  Subir ejecutable

 

El objetivo de la misma es proveer un medio para que el usuario ingrese un binario sobre el que se puede ejecutar un experimento.

Para esto se debe ingresar una descripción del programa, nombre de la carpeta en que se quiere guardar el programa y el archivo binario. El archivo debe ser un ejecutable. Si la carpeta en que se quiere contener el programa existe para algún programa previo, se sustituirán los contenidos de la carpeta anterior por los de este nuevo programa. Esto incluye carpetas de instalaciones.

 

Cuando se selecciona la opción Configurar nuevo experimento desde el panel de navegación, se visualizará la pantalla que se muestra en la figura 5.5:

Figura 5.5  Crear experimento

 

El objetivo de la misma es proveer un medio para que el usuario pueda configurar experimentos que puede ejecutar posteriormente.

Para esto se debe ingresar el nombre del experimento, una descripción del programa, seleccionar el programa sobre el que se ejecuta el experimento, indicar si el experimento tendrá que ejecutarse en paralelo o no, línea de comandos que recibirá el programa, y archivo de entrada en caso de que lo necesite. Además se podrán agregar al experimento parámetros de entrada, cada uno con un tipo, valor y nombre. Estos parámetros son utilizados para sustituir sus nombres por sus valores en la línea de comandos. Esto con ayuda de la siguiente sintaxis: $(NombreDelParametro). Es importante notar que los parámetros no deben tener espacios. Si el experimento está siendo configurado para que se pueda ejecutar en paralelo también se debe indicar si se debe guardar salidas para cada nodo, cuál es su directorio compartido de trabajo, el middleware que se está utilizando y la cantidad de procesadores a utilizar.

 

Cuando se quiere ver los detalles de un experimento, ver estadísticas o salidas de ejecuciones previas, o iniciar una nueva ejecución del experimento, se visualizará la pantalla que se muestra en la figura 5.6:

Figura 5.6 Ver detalle de experimento

 

El objetivo de la misma es desplegar la información configurada para el experimento (aquella con la que fue creado el experimento), mostrar las estadísticas y resultados de ejecuciones anteriores para que las puedan descargar y dar la opción para iniciar o detener la ejecución del experimento.

La información desplegada es aquella con que se creó el experimento. También hay una tabla de las ejecuciones hechas anteriormente con la fecha en que se realizaron y la opción de descargar sus estadísticas y resultados por medio de hipervínculos. Además al final de la información básica del experimento hay un botón que despliega la acción a realizar según el estado del experimento. Si el botón dice Detener es que el experimento está siendo ejecutado y que el usuario puede detenerlo en cualquier momento. De lo contrario dirá Iniciar para que el usuario inicie la ejecución del experimento.

En la pantalla que se muestra en la figura 5.7 podemos ver en mayor detalle a tabla de parámetros de entrada, la de corridas y la de contenidos del directorio.

6.  Barra de Menú

La única barra de menú en el sistema esta descrita por la siguiente imagen:

 

A continuación describimos la funcionalidad de cada una de las opciones del menú:

  • Mis ejecutables: Navegar a la página donde se despliegan todos los programas instalados y ejecutables subidos por el usuario.
  • Mis experimentos: Navegar a la página donde se despliegan todos los experimentos configurados por el usuario.
  • Configurar nuevo experimento: Navegar a la página donde el usuario puede configurar un nuevo experimento que posteriormente se desplegará en la página de Mis experimentos.

  • Instalar programa: Navegar a la página donde el usuario puede instalar nuevo programa que posteriormente se desplegará en la página de Mis ejecutables.
  • Lista de usuarios: Esta opción solo se desplegará a los usuarios administradores. Sirve para navegar a la página donde se listan todos los usuarios habilitados en el sistema además de su tipo de usuario. Desde esta página también se puede editar usuarios existentes.
  • Agregar usuario: Esta opción solo se desplegará a los usuarios administradores. Sirve para navegar a la página donde se puede registrar nuevos usuarios al sistema.

8.  Descripción de Funcionalidades

8.1 Instalar programa

Descripción:

Para ejecutar esta funcionalidad:

-   El usuario debe autenticarse dentro del sistema.

-   El usuario debe tener el programa en un archivo comprimido en los formatos permitidos por el sistema (tar.gz) o tiene un repositorio desde donde se pueden obtener los archivos fuente del programa.

Instrucciones:

1.    Seleccione la opción Instalar programa en el panel de navegación a la izquierda de la pantalla.

2.    Ingrese la descripción del programa y la carpeta destino. Si la carpeta destino que ingresa ya existe, esta va a ser reemplazada por el nuevo programa que está instalando. También indique el tipo de fuente de origen del programa, utilizando el check box. Los tipos de fuente del programa son por medio de un archivo local o un repositorio. En caso de ser un repositorio debe incluir la ruta completa (URL) en donde se encuentra el repositorio en el campo de texto donde se le es solicitado. Si es un archivo local debe seleccionarlo mediante el botón Seleccionar archivo y navegar dentro del explorador que le aparecerá hasta encontrar el archivo local. Este archivo debe ser de extensión tar.gz.

3.    Una vez que haya llenado toda la información para la instalación del programa, seleccione el botón Guardar.

 

8.2 Correr experimento

Descripción:

Para ejecutar esta funcionalidad:

-   El usuario debe autenticarse dentro del sistema.

-   El usuario debe tener al menos un experimento configurado.

Instrucciones:

1.    Selecciona la opción de Mis experimentos en el panel de navegación a la izquierda de la pantalla.

2.    Seleccione el hipervínculo Ver detalles del experimento que desea ejecutar o del que desea ver la información de configuración y sus ejecuciones previas.

3.    Seleccione el botón para iniciar la ejecución del programa. Si el programa ya está en ejecución entonces puede seleccionar el botón para detener el programa. Este se encuentra al final de la información básica del experimento, que está en el cuadro inicial de la página. Es importante destacar que la página no refresca cuando se termina la ejecución del programa por lo que se debe refrescar la página si se quiere saber el estado actual del experimento.

4.    Una vez que se termine la ejecución del experimento las estadísticas y salidas generadas por esta ejecución pueden verse al final de la tabla de información de ejecuciones en esta misma página.

8.3 Crear nuevo experimento

Descripción:

Para ejecutar esta funcionalidad:

-   El usuario debe autenticarse dentro del sistema.

-   El usuario debe haber subido o instalado al menos un programa previamente.

Instrucciones:

1.    Seleccione la opción de Configurar nuevo experimento en el panel de navegación a la izquierda de la pantalla.

2.    Ingrese el nombre del experimento, la descripción, seleccione el ejecutable sobre el que el experimento se va a ejecutar e indique si el experimento va a ser configurado para ejecutarlo en paralelo. En caso de que la ejecución sea en paralelo debe seleccionar la cantidad de procesadores a utilizar, si desea guardar las salidas generadas por cada nodo, el directorio compartido de trabajo y el middleware a utilizar. Todos los campos son obligatorios. También puede agregar opcionalmente una línea de parámetros de entrada del programa y un archivo de entrada.

3.    En la tabla de inferior que despliega los parámetros de entrada puede agregar nombres de parámetros que serán mapeados en la línea de comandos. Debe ingresar el nombre, el valor y seleccionar el tipo de parámetro para cada parámetro que desee ingresar. En la línea de comandos cada aparición de la siguiente expresión: $(nombre) será reemplazada por el valor del parámetro a la hora de ejecutar el programa sobre el que se basa el experimento.

4.    Seleccione la opción para guardar el nuevo experimento.

 

8.4 Caso de Uso UC4: Subir ejecutable

Descripción:

Para ejecutar esta funcionalidad:

-   El usuario debe autenticarse dentro del sistema.

-   El usuario debe tener en su computadora un archivo binario para usar como ejecutable.

Instrucciones:

1.    Seleccione la opción Subir ejecutable en el panel de navegación a la izquierda de la pantalla.

2.    Ingrese la descripción del programa y la carpeta destino. Si la carpeta destino que ingresa ya existe, esta va a ser reemplazada por el nuevo programa que está subiendo al sistema. Seleccione el botón Seleccionar archivo y navegue dentro del explorador que le aparecerá hasta encontrar el archivo binario que desea subir.

3.    Una vez que haya llenado toda la información para la instalación del programa, seleccione el botón Guardar.

 

8.5 Obtener resultados de experimento

Descripción:

Para ejecutar esta funcionalidad:

-   El usuario debe autenticarse dentro del sistema.

-   El usuario debe tener al menos un experimento configurado.

Instrucciones:

1.    Selecciona la opción de Mis experimentos en el panel de navegación a la izquierda de la pantalla.

2.    Seleccione el hipervínculo Ver detalles del experimento que desea ejecutar o del que desea ver la información de configuración y sus ejecuciones previas (incluyendo estadísticas y resultados).

3.    Vaya hasta el final de la página donde podrá ver una tabla con la información de ejecuciones realizadas previamente. Seleccione el hipervínculo que dice Descargar resultados en la fila del experimento para el que desea observar los resultados. Se le desplegará una ventana para escoger en donde desea guardar el archivo comprimido que contiene los resultados de la ejecución seleccionada. Descomprima el archivo y dentro encontrará archivos de texto con las salidas del programa.

8.6 Obtener estadísticas

Descripción:

Para ejecutar esta funcionalidad:

-   El usuario debe autenticarse dentro del sistema.

-   El usuario debe tener al menos un experimento configurado.

Instrucciones:

1.    Selecciona la opción de Mis experimentos en el panel de navegación a la izquierda de la pantalla.

2.    Seleccione el hipervínculo Ver detalles del experimento que desea ejecutar o del que desea ver la información de configuración y sus ejecuciones previas (incluyendo estadísticas y resultados).

3.    Vaya hasta el final de la página donde podrá ver una tabla con la información de ejecuciones realizadas previamente. Seleccione el hipervínculo que dice Descargar estadísticas en la fila del experimento para el que desea observar los resultados. Se le desplegará una ventana para escoger en donde desea guardar el archivo que contiene las estadísticas de la ejecución seleccionada.
